## Seção 1 – IDEs e editores de código
> Categoria obrigatória

**Ferramentas comparadas:**
- Ferramenta 1: `Visual Studio Code`
- Ferramenta 2: `WebStorm`
- Ferramenta 3: `Sublime Text`
- Ferramenta 4: `IntelliJ IDEA`

**Introdução:**
_Um ambiente de desenvolvimento integrado (IDE) é uma aplicação de software que ajuda os programadores a desenvolver código de software,como por exemplo o desenvolvimento web, de maneira eficiente. Ele aumenta a produtividade do desenvolvedor, combinando recursos como edição, compilação, teste e empacotamento de software em uma aplicação fácil de usar. Assim como escritores usam editores de texto e contadores usam planilhas, desenvolvedores de software usam IDEs para facilitar o trabalho._

| Suporte a JavaScript e Node.js | Excelente | Excelente | Bom | Muito bom |
| Suporte a HTML e CSS | Excelente | Excelente | Bom | Muito bom |
| Debugger integrado | Sim | Sim | Não nativo | Sim |
| Terminal integrado | Sim | Sim | Não | Sim |
| Integração com Git | Excelente | Excelente | Básica (plugins) | Excelente |
| Extensões disponíveis | Muitas | Algumas | Muitas | Muitas |
| Performance | Muito boa | Boa | Excelente | Boa |
| Custo / licença | Gratuito | Pago/ Gratuituo limitado | Pago/ Gratuituo | Pago / 30 dias grátis |
| **Nota geral (1 a 5)** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |

**Ferramenta escolhida:** `Visual Studio Code`

**Justificativa:**

Entre as ferramentas analisadas, a que mais se destacou foi o Visual Studio Code. Um dos principais motivos é que ele consegue reunir muitos recursos importantes sem ser pesado ou difícil de usar.

Outro ponto positivo é que ele é gratuito e possui uma grande variedade de extensões que podem ser instaladas conforme a necessidade do desenvolvedor. Isso permite adaptar o ambiente para trabalhar com JavaScript, Node.js, HTML, CSS e outras tecnologias usadas no desenvolvimento web.

Apesar de ferramentas como WebStorm e IntelliJ IDEA possuírem recursos muito avançados, elas são pagas e podem exigir mais do computador. Já o Sublime Text é muito rápido e leve, porém não possui tantas funcionalidades integradas.

Por isso, o Visual Studio Code acabou se destacando por oferecer um ambiente completo, fácil de usar e acessível para diferentes tipos de projetos, além de ser uma ferramenta á qual estamos mais habituados.

## Seção 2 – Clientes de banco de dados
> Categoria obrigatória

**Ferramentas comparadas:**
- Ferramenta 1: `PostgreSQL`
- Ferramenta 2: `SQLite3`
- Ferramenta 3: `MySQL`
- Ferramenta 4: `Microsoft SQL Server`

**Introdução:**
_Um sistema de gerenciamento de banco de dados (SGBD) é uma ferramenta utilizada para armazenar, organizar e gerenciar dados de forma estruturada. Em projetos de desenvolvimento de software, principalmente em aplicações web, o banco de dados é essencial para guardar informações importantes do sistema, como usuários, produtos e registros de atividades. Utilizar um bom sistema de banco de dados facilita a criação, consulta e manutenção dessas informações, além de garantir maior segurança e eficiência no acesso aos dados._

**Tabela comparativa:**

| Característica | Ferramenta 1 | Ferramenta 2 | Ferramenta 3 | Ferramenta 4 |
|---|---|---|---|---|
| Facilidade de instalação | Médio | Muito fácil | Fácil | Médio |
| Visualização de tabelas | Excelente | Boa | Excelente | Excelente |
| Editor de queries SQL | Excelente | Bom | Excelente | Excelente |
| Integração com a IDE | Muito boa | Boa | Muito boa | Excelente |
| Custo / licença | Gratuito / open source | Gratuito / open source | Gratuito / open source | Pago / versão Express gratuita |
| **Nota geral (1 a 5)** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

**Ferramenta escolhida:** `PostgreSQL`

**Justificativa:**

Entre as ferramentas analisadas, a que mais se destacou foi o PostgreSQL. Isso acontece porque ele oferece um conjunto muito completo de recursos para gerenciamento de bancos de dados, sendo bastante utilizado tanto em projetos acadêmicos quanto em sistemas profissionais.

Um dos principais pontos positivos do PostgreSQL é que ele é gratuito e open source, além de possuir grande estabilidade e confiabilidade no armazenamento e manipulação de dados. Ele também oferece suporte avançado para consultas SQL, relacionamentos complexos entre tabelas e diversas funcionalidades que ajudam no desenvolvimento de aplicações mais robustas.

Comparado com o SQLite3, por exemplo, o PostgreSQL possui mais recursos e é mais indicado para sistemas maiores. Já o MySQL também é bastante utilizado, porém o PostgreSQL costuma oferecer maior flexibilidade e mais funcionalidades avançadas. O Microsoft SQL Server é uma ferramenta muito poderosa, mas possui versões pagas e costuma ser mais utilizada em ambientes corporativos.

Dessa forma, o PostgreSQL se destacou por ser uma solução completa, gratuita e confiável para gerenciamento de bancos de dados em projetos de desenvolvimento de software.

## Seção 3 – Ferramentas de teste de API
> Categoria obrigatória

**Ferramentas comparadas:**
- Ferramenta 1: `Postman`
- Ferramenta 2: `Insomnia`
- Ferramenta 3: `Thunder Client`
- Ferramenta 4: `Hoppscotch`

**Introdução:**
_Ferramentas de teste de API são utilizadas para enviar requisições HTTP e analisar as respostas retornadas por servidores ou aplicações. Elas são muito importantes no desenvolvimento de software, especialmente em projetos que utilizam JavaScript e Node.js, pois permitem testar endpoints, validar dados enviados e recebidos, além de ajudar na identificação de erros durante o desenvolvimento. Essas ferramentas facilitam o trabalho do desenvolvedor ao permitir a simulação de diferentes tipos de requisições, como GET, POST, PUT e DELETE, de forma simples e organizada._

**Tabela comparativa:**

| Característica | Ferramenta 1 | Ferramenta 2 | Ferramenta 3 | Ferramenta 4 |
|---|---|---|---|---|
| Facilidade de uso | Muito fácil | Fácil | Muito fácil | Fácil |
| Suporte a métodos HTTP (GET, POST, PUT, DELETE) | Excelente | Excelente | Excelente | Excelente |
| Visualização da resposta | Excelente | Excelente | Muito boa | Muito boa |
| Organização de requisições | Excelente | Muito boa | Boa | Boa |
| Integração com a IDE | Boa | Boa | Excelente | Básica |
| Custo / licença | Gratuito / versão paga | Gratuito / versão paga | Gratuito / versão paga | Gratuito / versão paga |
| **Nota geral (1 a 5)** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |

**Ferramenta escolhida:** `Postman`

**Justificativa:**

Entre as ferramentas analisadas, a que mais se destacou foi o Postman. Isso ocorre porque ele é uma das ferramentas mais populares e completas para teste de APIs, sendo amplamente utilizada por desenvolvedores em diferentes tipos de projetos.

Um dos principais pontos positivos do Postman é sua interface intuitiva, que facilita o envio de requisições HTTP e a visualização das respostas do servidor. A ferramenta também permite organizar requisições em coleções, o que ajuda bastante durante o desenvolvimento e os testes de sistemas que possuem várias rotas de API.

Além disso, o Postman oferece suporte completo aos principais métodos HTTP, como GET, POST, PUT e DELETE, permitindo testar praticamente qualquer tipo de endpoint de uma aplicação. Ele também possui recursos adicionais, como testes automatizados, variáveis de ambiente e compartilhamento de coleções entre membros de uma equipe.

Comparado às outras ferramentas analisadas, como Insomnia, Thunder Client e Hoppscotch, o Postman possui um conjunto de funcionalidades mais completo e uma comunidade muito grande de usuários.

Por esses motivos, o Postman se destacou como a ferramenta mais completa e amplamente utilizada para testes de APIs em projetos de desenvolvimento.

## Seção 4 – Categoria complementar

**Categoria escolhida:** `Qualidade de código`

**Ferramentas comparadas:**
- Ferramenta 1: `ESLint`
- Ferramenta 2: `Prettier`
- Ferramenta 3: `JSHint`
- Ferramenta 4: `SonarQube`

**Introdução:**
_Ferramentas de qualidade de código são utilizadas para analisar, padronizar e melhorar o código desenvolvido durante um projeto de software. Elas ajudam os desenvolvedores a identificar erros, más práticas de programação e inconsistências na formatação do código. No desenvolvimento com JavaScript, essas ferramentas são muito importantes para manter o código mais organizado, legível e fácil de manter ao longo do tempo. Além disso, ajudam equipes de desenvolvimento a seguir padrões de codificação, reduzindo a chance de bugs e melhorando a colaboração entre programadores._

**Tabela comparativa:**

| Característica | Ferramenta 1 | Ferramenta 2 | Ferramenta 3 | Ferramenta 4 |
|---|---|---|---|---|
| Facilidade de uso | Fácil | Muito fácil | Fácil | Médio |
| Integração com a IDE | Excelente | Excelente | Boa | Boa |
| Documentação e comunidade | Excelente | Excelente | Boa | Muito boa |
| Custo / licença | Gratuito / open source | Gratuito / open source | Gratuito / open source | Gratuito / versão paga |
| **Nota geral (1 a 5)** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |

**Ferramenta escolhida:** `ESLint`

**Justificativa:**

Entre as ferramentas analisadas, a que mais se destacou foi o ESLint. Essa ferramenta é amplamente utilizada no desenvolvimento em JavaScript e possui como principal objetivo analisar o código e identificar possíveis erros, problemas de sintaxe ou padrões inadequados de programação.

Um dos principais pontos positivos do ESLint é a sua grande capacidade de personalização, permitindo que os desenvolvedores definam regras específicas para manter um padrão de qualidade no código do projeto. Além disso, ele possui excelente integração com diversas IDEs, como o Visual Studio Code, o que facilita muito o uso durante o desenvolvimento.

Comparado com outras ferramentas analisadas, como o Prettier e o JSHint, o ESLint possui um sistema mais completo de análise de código. Já o SonarQube oferece recursos avançados de análise, mas costuma ser mais utilizado em ambientes corporativos e pode exigir uma configuração mais complexa.

Dessa forma, o ESLint se destacou por ser uma ferramenta gratuita, muito popular e eficiente para manter a qualidade e organização do código em projetos JavaScript.
